# Staging environment setup

Staging is a parallel Railway service pointed at a `staging` git branch
and a separate Turso database. Keep it architecturally identical to
prod — different data, same code, same infrastructure shape.

## One-time Railway wire-up

1. In the existing Railway project → **New → Deploy from GitHub**.
2. Pick the same repo, set **Branch** = `staging`, **Root directory** =
   repo root (default).
3. Name the service `flow-procurement-staging`.
4. **Variables** — copy prod but override:
   - `FLOW_SENTRY_ENV=staging` (so Sentry separates issues)
   - `FLOW_TURSO_DATABASE_URL` → new staging Turso DB (do not share
     prod data)
   - `FLOW_TURSO_AUTH_TOKEN` → new token scoped to the staging DB
   - `FLOW_JWT_SECRET` → separate random value (rotating prod must
     not log users out of staging)
   - `FLOW_LLM_API_KEY` — can share with prod; the LLM usage is small
   - `FLOW_RATE_LIMIT_PER_MINUTE=300` — looser for load testing
5. Deploy. First build matches prod timing.

## Git branch flow

```
feature-branch  →  PR into staging  →  auto-deploy to staging
                                    → manual PR into main  →  auto-deploy to prod
```

Pre-commit hook bumps PATCH version on each commit. Staging and prod
will share version numbers — the Sentry `environment` tag keeps them
distinguishable.

## Staging smoke test

After every staging deploy:
```bash
BASE_URL=https://flow-procurement-staging.up.railway.app k6 run ops/load-test.js
```

k6 thresholds: p95<1.5s, p99<3s, error_rate<1%. Fail → don't merge
into main.

## Promote staging → prod

```bash
# On main
git checkout main
git merge staging --ff-only   # or open a PR for review
git push origin main
```

Railway prod picks up automatically. Watch for 5-8 min build + 30s
health check.

## Data strategy

- **Users**: staging starts with the demo seed (`admin/admin123`
  etc.). Real test users land in staging via the admin UI.
- **Contracts**: staging auto-seeds the 5 demo contracts on first
  read. Real contracts land via `POST /api/v1/buying/contracts`.
- **Orders**: regenerated by the checkout flow during testing. No
  cross-environment copy.

## Cost

Railway bills per service. At current tier (~$5/mo per replica)
staging doubles the platform cost. Turso's free tier covers a
second small DB. Budget: ~$10-15/mo total for staging + prod.
